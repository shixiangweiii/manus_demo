## 推理引擎类型与环境变量配置指南

系统共有 **4 种推理引擎**（v1 Simple / v2 Complex / v5 Emergent / v8 Goal-Driven），由 `PLAN_MODE` + `ENABLE_GOAL_DRIVEN_PLANNER` 两个核心变量组合决定执行路径。另外还有 **v9 SubAgent** 子智能体机制，可在任意 ReAct 循环路径中以工具形式调用。

---

## 一、核心路由变量

| 变量 | 默认值 | 作用 |
|------|--------|------|
| `PLAN_MODE` | `auto` | 决定走 simple / complex / emergent 路径 |
| `ENABLE_GOAL_DRIVEN_PLANNER` | `false` | 在 emergent 路径内决定走 v5 还是 v8 |
| `EMERGENT_PLANNING_ENABLED` | `true` | emergent 总开关；关闭时所有 emergent 分类降级为 complex |

---

## 二、各引擎详解与环境变量配置

### 1. Simple — v1 扁平计划（顺序执行）

最轻量的执行路径。PlannerAgent 将任务分解为 2-6 个扁平步骤，按顺序逐个通过 ReAct 循环执行，最后整体反思。

**必选配置：**
```bash
PLAN_MODE=simple
```

**执行路径：**
```
User Task → Planner.create_plan() → Executor (逐 Step ReAct) → Reflector.reflect()
  ↓ 反思失败
  Replan (最多 MAX_REPLAN_ATTEMPTS 次) → 重新执行
```

**相关调优变量：**

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `MAX_REPLAN_ATTEMPTS` | `3` | 反思失败后最大重规划次数 |
| `MAX_REACT_ITERATIONS` | `10` | 每个 Step 内部 ReAct 循环最大迭代次数 |
| `MAX_CONTEXT_TOKENS` | `8000` | 上下文 Token 上限，超出后触发摘要压缩 |
| `TOKEN_TRACKING_ENABLED` | `true` | 是否启用 Token 消耗追踪 |
| `ENABLE_REACT_ENGINE_V2` | `false` | 是否使用 v6 统一 ReActEngine（不影响路由，仅切换内部执行引擎） |
| `LLM_RETRY_ENABLED` | `false` | LLM 调用失败时是否自动重试 |

> `ENABLE_GOAL_DRIVEN_PLANNER` 在此路径下不生效（emergent 路径未进入）。

---

### 2. Complex — v2 DAG（分层并行超步执行）

中等复杂度路径。PlannerAgent 生成三层分层 DAG（Goal → SubGoals → Actions），由 DAGExecutor 按拓扑序以 Super-step 模型执行，支持并行执行、条件分支、回滚、自适应规划。

**必选配置：**
```bash
PLAN_MODE=complex
```

**执行路径：**
```
User Task → Planner.create_dag() → DAGExecutor (Super-step 循环)
  → Reflector.reflect_dag()
  ↓ 反思失败
  Planner.replan_subtree() (局部子树重规划，保留已完成节点) → 重新执行
```

**相关调优变量：**

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `MAX_PARALLEL_NODES` | `3` | 每个 Super-step 最多并行执行的节点数 |
| `DAG_SERIAL_EXECUTION` | `true` | 是否串行执行 DAG 节点（默认串行；设 `false` 恢复并行） |
| `ADAPTIVE_PLANNING_ENABLED` | `true` | 是否启用超步间自适应规划（根据中间结果动态调整待执行节点） |
| `ADAPT_PLAN_INTERVAL` | `1` | 每隔几个超步执行一次自适应检查（1 = 每步都检查） |
| `ADAPT_PLAN_MIN_COMPLETED` | `1` | 至少完成多少个 ACTION 节点后才启动自适应 |
| `NODE_EXECUTION_TIMEOUT` | `300` | 单个节点执行超时（秒） |
| `MAX_CHECKPOINTS` | `10` | 内存中保留的最大 Checkpoint 数量 |
| `TOOL_FAILURE_THRESHOLD` | `2` | 连续失败多少次后 ToolRouter 建议切换工具 |
| `MAX_REPLAN_ATTEMPTS` | `3` | 反思失败后最大重规划次数 |
| `MAX_REACT_ITERATIONS` | `10` | 每个 Action 节点内 ReAct 循环最大迭代次数 |
| `ENABLE_REACT_ENGINE_V2` | `false` | 是否使用 v6 统一 ReActEngine |

**并行安全说明：** 并行模式（`DAG_SERIAL_EXECUTION=false`）通过 `create_for_node()` 为每个并行节点创建独立的 `ExecutorAgent` 实例，避免 `_messages` 竞态条件；`DAGState.node_results` 使用 per-node dict key，无写冲突。

---

### 3. Emergent v5 — TODO 列表驱动（Claude Code 风格）

隐式规划路径。无独立规划阶段，规划通过 TODO 列表管理自然涌现：LLM 在 ReAct 循环中动态创建、更新、完成 TODO 项。

**必选配置：**
```bash
PLAN_MODE=emergent
ENABLE_GOAL_DRIVEN_PLANNER=false
```

> `EMERGENT_PLANNING_ENABLED` 默认为 `true`，一般无需显式设置。若设为 `false`，emergent 分类会降级为 complex。

**执行路径：**
```
User Task → EmergentPlannerAgent.execute()
  → 初始化 TODO (1-3 项)
  → while (has_pending_todos && iteration < MAX_EMERGENT_OUTER_ITERATIONS):
      → 选择下一个 ready TODO
      → think_with_tools() (ReAct 推理 + 工具调用)
      → 成功: 标记完成, 可选更新 TODO 列表
      → 失败: 重试 (最多 MAX_TODO_RETRIES 次), 然后标记 BLOCKED
  → 汇编最终答案
```

**相关调优变量：**

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `MAX_TODO_ITEMS` | `20` | TODO 列表最大项数 |
| `MAX_TODO_RETRIES` | `3` | 单个 TODO 最大重试次数 |
| `TODO_COMPRESSION_THRESHOLD` | `0.8` | 上下文窗口使用率达 80% 时压缩 TODO |
| `MAX_EMERGENT_OUTER_ITERATIONS` | `60` | Emergent 主循环最大迭代数（默认 = MAX_TODO_ITEMS × MAX_TODO_RETRIES） |
| `MAX_REACT_ITERATIONS` | `10` | 每个 TODO 内部 ReAct 最大迭代次数 |
| `ENABLE_REACT_ENGINE_V2` | `false` | 是否使用 v6 统一 ReActEngine（控制 v5 内部执行引擎） |
| `MAX_CONTEXT_TOKENS` | `8000` | 上下文 Token 上限 |
| `TOOL_FAILURE_THRESHOLD` | `2` | 连续失败多少次后建议切换工具 |

**停滞检测：** 若连续 3+ 轮无 TODO 完成，自动提前终止。

---

### 4. Emergent v8 — 目标驱动规划（GoalDrivenPlanner）

「以终为始」执行引擎，受 ReflAct（EMNLP 2025）启发。在 v5 基础上增加持久化 GoalDocument、逆向里程碑规划、目标反思与重锚定机制，防止长流程任务中的目标漂移。

**必选配置：**
```bash
PLAN_MODE=emergent
ENABLE_GOAL_DRIVEN_PLANNER=true
```

**执行路径：**
```
User Task → GoalDrivenPlannerAgent.execute()
  → 构建 GoalDocument (定义 "完成标准")
  → 逆向规划: 从终态推导 Milestones
  → Milestones → TodoList
  → while (has_pending && iteration < MAX_GOAL_DRIVEN_ITERATIONS):
      → GoalReflection: 对比当前状态 vs 目标
      → 选择受反思引导的 TODO
      → 执行 TODO (目标注入的 ReAct 循环)
      → 每 GOAL_REANCHOR_INTERVAL 轮重新锚定目标文档
  → 基于 GoalDocument 汇编最终答案
```

**相关调优变量：**

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `GOAL_REANCHOR_INTERVAL` | `5` | 每隔多少次外层迭代重新锚定目标文档 |
| `GOAL_REFLECTION_INTERVAL` | `1` | 每隔多少次外层迭代执行目标反思（1 = 每次都反思） |
| `MAX_GOAL_DRIVEN_ITERATIONS` | `60` | v8 主循环最大迭代数（默认 = MAX_TODO_ITEMS × MAX_TODO_RETRIES） |
| `GOAL_DRIVEN_STAGNATION_WINDOW` | `3` | 连续多少轮无进度突破则提前终止 |
| `MAX_TODO_ITEMS` | `20` | TODO 列表最大项数 |
| `MAX_TODO_RETRIES` | `3` | 单个 TODO 最大重试次数 |
| `MAX_REACT_ITERATIONS` | `10` | 每个 TODO 内 ReAct 最大迭代次数 |
| `MAX_CONTEXT_TOKENS` | `8000` | 上下文 Token 上限 |

### 5. Auto — 自动分类（默认模式）

两阶段混合分类器自动选择最优引擎。规则快筛处理 60-70% 的显然任务（零成本），仅模糊区间触发 LLM 分类调用。

**配置：**
```bash
PLAN_MODE=auto    # 默认值，可省略
```

**分类决策：**
- **Stage 1 — 规则快筛**（零成本，< 1ms）：基于任务长度、多步关键词、条件/并行模式、动作动词密度、探索性/不确定性模式打分
- **Stage 2 — LLM 分类**（仅 Stage 1 返回 ambiguous 时触发）：轻量调用（~60 tokens），temperature=0.0，失败时默认降级为 complex

**路由规则：**
```
分类结果 "simple"   → v1 扁平计划
分类结果 "complex"  → v2 DAG 并行
分类结果 "emergent" → ENABLE_GOAL_DRIVEN_PLANNER=true ? v8 : v5
```

**降级规则：**
- `EMERGENT_PLANNING_ENABLED=false` 时，所有 emergent 分类（规则或 LLM 给出）自动降级为 complex
- 未知分类结果降级为 complex
- LLM 分类调用失败降级为 complex

## 三、v9 SubAgent 子智能体（可选增强，跨引擎可用）

SubAgent 遵循 Claude Code Subagent 模式，可在任意 ReAct 循环路径（simple/complex/emergent）中由 LLM 主动调用 `subagent` 工具来委派复杂子任务。深度限制为 1（不能嵌套）。

**必选配置：**
```bash
SUBAGENT_ENABLED=true
```

**典型组合：**
```bash
# Emergent + SubAgent
SUBAGENT_ENABLED=true PLAN_MODE=emergent python main.py "多步骤研究任务"

# Complex + SubAgent
SUBAGENT_ENABLED=true PLAN_MODE=complex python main.py "复杂分析任务"
```

**相关调优变量：**

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `SUBAGENT_ENABLED` | `false` | 总开关（默认关闭） |
| `SUBAGENT_MAX_ITERATIONS` | `10` | SubAgent 内部 ReAct 最大迭代次数（默认 = MAX_REACT_ITERATIONS） |
| `SUBAGENT_TIMEOUT` | `300` | SubAgent 执行超时（秒，默认 = NODE_EXECUTION_TIMEOUT） |
| `SUBAGENT_MAX_CONCURRENT` | `3` | 最大并发 SubAgent 数量 |
| `SUBAGENT_SUMMARY_MAX_LENGTH` | `2000` | SubAgent 返回摘要最大字符数 |
| `SUBAGENT_MAX_CALLS_PER_TASK` | `3` | 单任务 SubAgent 调用次数上限（防反模式 #3/#8） |
| `SUBAGENT_MAX_TOKENS_PER_CALL` | `50000` | 单次 SubAgent 调用 Token 预算上限（防 Token 爆炸） |
| `SUBAGENT_DEFAULT_TOOL_WHITELIST` | `""` | 默认工具白名单（逗号分隔，空 = 全量授权） |

**安全机制：**
- **depth=1 结构性强制**：SubAgent 的工具白名单中不包含 `subagent` 工具，LLM 物理上无法嵌套调用
- **独立上下文**：独立的 messages 列表、系统提示词、ReActEngine 实例
- **摘要返回**：父级仅接收 `SubAgentSummary` JSON，不暴露完整对话
- **Token 预算熔断**：通过 `on_iteration` 回调实时检测累计 Token，超限抛出 `SubAgentTokenExhausted`

## 四、完整决策树

```
PLAN_MODE=simple   ─────────────────────→ v1 扁平计划
PLAN_MODE=complex  ─────────────────────→ v2 DAG 并行
PLAN_MODE=emergent
    ├── EMERGENT_PLANNING_ENABLED=false ──→ 降级为 v2 DAG
    ├── ENABLE_GOAL_DRIVEN_PLANNER=true  ──→ v8 目标驱动
    └── ENABLE_GOAL_DRIVEN_PLANNER=false ──→ v5 TODO 列表
PLAN_MODE=auto
    ├── 规则/LLM 分类为 "simple"    ──→ v1 扁平计划
    ├── 规则/LLM 分类为 "complex"   ──→ v2 DAG 并行
    └── 规则/LLM 分类为 "emergent"
        ├── EMERGENT_PLANNING_ENABLED=false ──→ 降级为 v2 DAG
        ├── ENABLE_GOAL_DRIVEN_PLANNER=true  ──→ v8 目标驱动
        └── ENABLE_GOAL_DRIVEN_PLANNER=false ──→ v5 TODO 列表
    └── 分类失败/未知 ──→ 降级为 v2 DAG

任意 ReAct 路径 + SUBAGENT_ENABLED=true
    └── LLM 可调用 "subagent" 工具 → SubAgent (depth=1, 独立上下文, 摘要返回)
```

## 六、常用启动命令速查

```bash
# 默认自动分类
python main.py "分析今天的天气并给出建议"

# 强制各引擎
PLAN_MODE=simple python main.py "简单任务"
PLAN_MODE=complex python main.py "复杂多阶段任务"
PLAN_MODE=emergent python main.py "探索性研究任务"

# v8 目标驱动
ENABLE_GOAL_DRIVEN_PLANNER=true PLAN_MODE=emergent python main.py "长流程分析任务"

# 启用 SubAgent
SUBAGENT_ENABLED=true PLAN_MODE=emergent python main.py "多步骤研究任务"

# DAG 并行模式（默认串行）
DAG_SERIAL_EXECUTION=false PLAN_MODE=complex python main.py "并行任务"

# 启用追踪
TRACING_ENABLED=true TRACING_BACKEND=rich python main.py "任务"

# 查看追踪
python -m tracing    # 启动 Web Viewer http://localhost:8000
```
