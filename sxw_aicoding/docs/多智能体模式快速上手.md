## 一、SubAgent 是什么？什么时候用？

SubAgent 遵循 Claude Code Subagent 模式：父 Agent（Executor / EmergentPlanner / GoalDrivenPlanner）在 ReAct 循环中可调用 `subagent` 工具派生**隔离式子智能体**完成复杂子任务。

### 适合 SubAgent 的子任务

子任务需**同时**满足以下三条:
1. **需要 ≥3 次工具调用**（多步工作）
2. **是自包含的工作单元**，边界清晰
3. **结果可被概括**，不需要父级看到全部中间步骤

典型场景：「调研 X 并写到 file Y」「在代码库里搜 Z 然后整理成清单」「批量处理 N 个文件，最后报告」。

### 不适合 SubAgent 的子任务

- 单次工具调用就能完成（读一个文件、跑一个命令、单次搜索）
- 父级需要看中间结果再决定下一步（比如交互式探索）
- 简单转换（字符串处理、数据格式变换）

> **判断规则**：拿不准时**直接用基础工具**。SubAgent 用上下文可见性换隔离性 —— 不必要的派生只会增加 token 与延迟。

## 二、5 分钟开启

### 最小启动命令

```bash
SUBAGENT_ENABLED=true python main.py "<your task>"
```

仅此一行。`SUBAGENT_ENABLED=true` 后，OrchestratorAgent 自动把 `subagent` 工具注入所有 ReAct 循环的工具列表，LLM 可在任何路径（simple / complex / emergent）按需自主调用。

### 推荐配置：emergent + GoalDriven + SubAgent（最常见）

```bash
SUBAGENT_ENABLED=true \
ENABLE_GOAL_DRIVEN_PLANNER=true \
PLAN_MODE=emergent \
python main.py "请用 subagent 派生子智能体写入 sandbox/hello.txt（内容 'world'），然后告诉我成功"
```

主路径走 v8 GoalDrivenPlanner（目标锚定 + ReflAct 反思），SubAgent 处理具体子任务。这是当前最有教学价值的组合。

### 给 SubAgent 限定工具白名单（推荐）

LLM 在调用 `subagent` 时可显式传 `tool_whitelist` 参数；如果没传，可用 config 默认值：

```bash
SUBAGENT_ENABLED=true \
SUBAGENT_DEFAULT_TOOL_WHITELIST=file_ops,web_search \
python main.py "复杂任务"
```

注意：**`subagent` 与 `ask_user` 永远会被结构性排除**（depth=1 + HITL 隔离），无需手动配置。

## 三、10 分钟看懂归因证据

### UI 实时观察

任务执行时，控制台会按事件渲染：

```
[SubAgent] SubAgent-1 spawned: 写入 sandbox/hello.txt（内容 'world'）...
   [dim][SubAgent][/dim] SubAgent-1 iter 2 (1 tool calls)        # 进度（Wave-4 L5: 默认每 N 轮）
   [dim][SubAgent][/dim] SubAgent-1 iter 4 (2 tool calls)
[SubAgent] SubAgent-1 completed (4 iters, 6377 tokens, 6836ms)
```

任务结束时,出现 by_caller token 表（**Wave-6 新增**）：

```
                Token Consumption by Caller (Wave-6)
┃ Caller / Agent           ┃  Prompt Tokens ┃  Completion ┃   Total Tokens ┃
│ GoalDrivenPlanner        │          25834 │        4258 │          30092 │
│ SubAgent-1               │           5349 │         751 │           6100 │
└──────────────────────────┴────────────────┴─────────────┴────────────────┘
```

主-子 token 完全分账。

### Trace 文件验证（强证据）

启用 file backend 持久化 trace：

```bash
SUBAGENT_ENABLED=true \
ENABLE_GOAL_DRIVEN_PLANNER=true \
PLAN_MODE=emergent \
TRACING_ENABLED=true \
TRACING_BACKEND=file \
python main.py "<task>"
```

完成后 `traces/` 目录会有最新 trace JSON。用 grep 验证两个关键属性：

```bash
# Wave-1 H1: SubAgent 父级归因
grep -o '"subagent.parent_agent":[^,]*' traces/<id>.json | sort -u
# 输出应为 GoalDrivenPlanner / EmergentPlanner / ExecutorAgent 等真实父级，
# 而非硬编码的 "OrchestratorAgent"

# Wave-6: LLM 调用按 caller 分组
grep -o '"gen_ai.caller":[^,]*' traces/<id>.json | sort | uniq -c
# 输出形如:
#   12 "gen_ai.caller": "GoalDrivenPlanner"
#    4 "gen_ai.caller": "SubAgent-1"
```

或用 web viewer：

```bash
python -m tracing    # http://localhost:8000
```

打开任意 trace，点开任意 `subagent.execute.*` span，右侧详情面板会显示 `subagent.parent_agent` / `subagent.tool_whitelist` / `subagent.iterations_used` 等。

### Python 一次性归因汇总（便于自动化检查）

```python
import json, glob, os
from collections import Counter

trace = sorted(glob.glob('traces/*.json'), key=os.path.getmtime)[-1]
data = json.load(open(trace))

caller_tokens = Counter()
parent_attribution = Counter()
for span in data.get('spans', []):
    name = span.get('name', '')
    attrs = span.get('attributes', {})
    if name.startswith('llm.'):
        caller_tokens[attrs.get('gen_ai.caller', '(missing)')] += attrs.get('gen_ai.usage.total_tokens', 0)
    if name.startswith('subagent.execute'):
        parent_attribution[attrs.get('subagent.parent_agent', '(missing)')] += 1

print(f'LLM by caller: {dict(caller_tokens)}')
print(f'SubAgent parents: {dict(parent_attribution)}')
```

## 四、配置参考

### SubAgent 专属

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `SUBAGENT_ENABLED` | `false` | 总开关 |
| `SUBAGENT_MAX_ITERATIONS` | `10` | SubAgent 内部 ReAct 最大迭代 |
| `SUBAGENT_TIMEOUT` | `300` | 单 SubAgent 超时（秒） |
| `SUBAGENT_MAX_CONCURRENT` | `3` | 最大并发 SubAgent |
| `SUBAGENT_MAX_CALLS_PER_TASK` | `3` | 单任务 SubAgent 派生上限（防 token 爆炸） |
| `SUBAGENT_MAX_TOKENS_PER_CALL` | `50000` | 单 SubAgent token 预算上限（熔断） |
| `SUBAGENT_SUMMARY_MAX_LENGTH` | `2000` | 返回摘要最大字符数 |
| `SUBAGENT_DEFAULT_TOOL_WHITELIST` | `""` | 默认工具白名单，逗号分隔 |
| `SUBAGENT_MAX_TASK_DESCRIPTION_LENGTH` | `2000` | **Wave-4** L2：超长任务描述截断阈值 |
| `SUBAGENT_ITERATION_EVENT_VERBOSITY` | `summary` | **Wave-4** L5：UI 渲染粒度 `summary`/`full`/`silent` |
| `SUBAGENT_ITERATION_EVENT_EVERY_N` | `2` | **Wave-4** L5：summary 模式每 N 轮渲染一次 |

### 与 SubAgent 协同的关键全局配置

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `MAX_REACT_ITERATIONS` | `10` | ReAct 最大迭代（影响 SubAgent 内部循环） |
| `MAX_CONTEXT_TOKENS` | `16000` | 上下文 token 上限（**注意**：v12 升 8000→16000，旧 .env 可能仍写 8000） |
| `TOOL_RESULT_TRUNCATION_LIMIT` | `2000` | 工具输出截断（影响 SubAgent 看到的 tool message 大小） |
| `SANDBOX_DIR` | `~/.manus_demo/sandbox` | SubAgent 子目录的父目录 |

## 五、常见组合速查

```bash
# 1. 最小：单纯启用 SubAgent，主路径自动分类
SUBAGENT_ENABLED=true python main.py "任务"

# 2. 教学推荐：v8 目标驱动 + SubAgent
SUBAGENT_ENABLED=true ENABLE_GOAL_DRIVEN_PLANNER=true PLAN_MODE=emergent \
  python main.py "任务"

# 3. 限定 SubAgent 只能用文件操作 + 网络搜索
SUBAGENT_ENABLED=true \

  python main.py "任务"

SUBAGENT_ENABLED=true ENABLE_GOAL_DRIVEN_PLANNER=true PLAN_MODE=emergent \
TRACING_ENABLED=true TRACING_BACKEND=file \
  python main.py "任务"

# 5. SubAgent + HITL：交互模式，SubAgent 不会调用 ask_user(已结构性排除),
```

## 六、故障排查

### 现象 1：SubAgent 没有被调用

**检查**：
1. `SUBAGENT_ENABLED=true` 是否真的设置（`echo $SUBAGENT_ENABLED`）
2. 任务是否需要 SubAgent —— 简单任务 LLM 不会主动派生（这是预期行为，见第一节"何时用"）
3. 任务描述里是否暗示派生（如显式说「请用 subagent...」）

**强诱导测试**：
```bash
SUBAGENT_ENABLED=true python main.py "请使用 subagent 工具派生一个子智能体写入 sandbox/test.txt 内容 hello"
```

### 现象 2：父级归因显示 "OrchestratorAgent" 而非 "GoalDrivenPlanner"（修复前的旧 bug）

**Wave-1 前**这是 H1 真实 bug —— GoalDrivenPlanner 不调 set_caller，SubAgent.parent_agent 恒为 SubAgentTool 构造时的默认值 `OrchestratorAgent`。

**Wave-1 后**(v9.1)已修复。如再次出现，应是新加的代码路径漏调 `attribute_caller(tool, agent_name)`。grep 验证：

```bash
grep -rn "attribute_caller\|set_caller" agents/ react/
```

确保自有 ReAct 循环（如 GoalDrivenPlanner._execute_todo_goal_guided）调用了 `attribute_caller(t, "AgentName")`。

### 现象 3：`subagent_<N>` 子目录残留

**Wave-4 M3 修复后**：每次 task_start 时 `OrchestratorAgent.run()` 调 `SubAgentTool.reset_task_state()`,自动清理 `SANDBOX_DIR/subagent_<digits>` 子目录。

```bash
# 手动清理（罕见情况）
rm -rf ~/.manus_demo/sandbox/subagent_*
```

注意：清理只针对 `subagent_<digits>` 模式的子目录,`SANDBOX_DIR` 根目录下的其他文件 / 用户数据 **不会**被动**。

### 现象 4：SubAgent 调用 limit 触发后,ToolRouter 提示"使用替代工具"

**Wave-1 H2 前**这是真实 bug —— GoalDrivenPlanner 的工具循环把 `Error: SubAgent call limit reached...` 算作 failure，几次累积触发 ToolRouter 的"使用替代工具"提示，但 SubAgent 没有"替代"。

**Wave-1 后**(v9.1)已通过 `classify_result()` 三态分类修复：rate_limited 单独成桶（`record_rate_limited`），不污染 failure 阈值。

### 现象 5：父级 LLM token 与 SubAgent token 混在一起，看不清谁花了多少

**Wave-6 前**：所有 token 都按 model（`engine` 字段）聚合，无 caller 维度。
**Wave-6 后**：UI 自动渲染 by_caller 表格 + trace 写入 `gen_ai.caller`。

如果某 caller 显示为 `unknown`,说明该 LLM 调用路径没有传 caller_tag。在 grep 找漏点：

```bash
grep -n "chat_with_tools\|chat_json\|client.chat" agents/ react/ | grep -v "caller_tag"
```

应只剩 BaseAgent 的内部封装（自动 setdefault `caller_tag=self.name`）。其它直接调 llm_client 的位置都应显式传 caller_tag。

### 现象 6：SubAgent 失败摘要丢了最后一轮 tool 调用

**Wave-3 M2 前**：失败路径用 `_accumulated_tool_calls`(on_iteration 末尾快照),timeout 中途触发会丢。
**Wave-3 M2 后**：`_failure_tool_calls()` 优先读 `self._react_engine._current_log`(实时 mid-iteration)。如再次出现，检查 `agents/subagent.py:_failure_tool_calls()` 实现是否还在。

### 现象 7：DAG 并行 + SubAgent 时 UI 进度信息冲淡了关键事件

调低渲染粒度：

```bash
SUBAGENT_ITERATION_EVENT_VERBOSITY=summary \
SUBAGENT_ITERATION_EVENT_EVERY_N=5 \
  python main.py "任务"
# 或干脆静默
SUBAGENT_ITERATION_EVENT_VERBOSITY=silent python main.py "任务"
```

## 七、与其他模块的协同

### 与评测模块（evaluation）

`evaluation/runner.py` 在 task 结束时优先按 `LLMCallRecord.caller_tag` 聚合 SubAgent token（Wave-7），事件聚合作为 fallback。`benchmark.py` 的部分 hard 任务设计为多步骤，自然触发 SubAgent；评测分数会反映 SubAgent 调用成功率与 token 效率。详见 [`evaluation-guide.md`](evaluation-guide.md)。

### 与 Tracing（v7）

每个 SubAgent 派生形成 `subagent.execute.<name>` span，下挂自己的 `llm.*` / `tool.execute.*` 子 span。`subagent.parent_agent` 属性记录派生方，`gen_ai.caller`（Wave-6）记录每次 LLM 调用的发起 agent。详见 [`tracing-guide.md`](tracing-guide.md)。

### 与 HITL（v13）

SubAgent 工具白名单**结构性排除** `ask_user`（3 处过滤位点：tools/subagent_tool.py），且系统提示也不注入 ask_user 引导（Wave-2 M1）—— SubAgent 必须通过 `SubAgentSummary.issues` 字段汇报歧义,**不能**直接向用户提问。
