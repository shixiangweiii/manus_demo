## 执行层

### 🧠 论文 03: CodeAct — Your LLM Agent Acts Better when Generating Code

| 字段 | 内容 |
|------|------|
| 作者 | Xingyao Wang, Yangyi Chen, Lifan Yuan 等 |
| 发表 | ICML 2024 (Apple Research) |
| 一句话定性 | **2024 年执行层最重要革新**：把"输出 JSON 工具调用"替换为"输出可执行 Python 代码" |

**核心贡献**:
- 把所有 action 统一到 Python 代码空间（sandbox 执行 + 返回 stdout/stderr/exception 作为 observation）
- 支持**一轮 LLM 输出内组合多工具**：循环、条件、变量传递、临时函数定义
- 与 ReAct 相比，**同样任务所需 LLM 轮次下降 50%+**
- 在 Apple 内部 API v2 和 Microsoft M365 Copilot 已落地

**方法要点**:
```python
# ReAct 方式（需 3 轮）
Thought: I need to search X
Action: {"tool": "search", "q": "X"}   <- 第1轮
Observation: [...]
Thought: Now I'll filter by year 2024
Action: {"tool": "filter", ...}         <- 第2轮
Observation: [...]
Thought: Summarize
Action: final_answer(...)               <- 第3轮

# CodeAct 方式（1 轮即可）
Thought: Search then filter then summarize
Code:
  results = search("X")
  filtered = [r for r in results if r.year == 2024]
  return summarize(filtered)            <- 一次搞定
```

**关键实验**:
- API-Bank: CodeAct 相对 JSON baseline 提升 20%
- MINT: 多步推理任务成功率提升明显
- 2025 年 Microsoft Agent Framework 正式支持

**对本项目启示**:
- 本项目已有 `tools/code_executor.py` + `tools/shell_tool.py`，基础设施齐全
- **v9 建议首先引入 CodeAct**，作为 `agents/codeact_executor.py` 新 agent
- 可设计对照实验：同 12 个基准任务，v8 (JSON) vs v8+CodeAct 的轮次/成功率/成本对比

**链接**:
- Apple Research: https://machinelearning.apple.com/research/codeact
- Microsoft 工程化: https://devblogs.microsoft.com/agent-framework/codeact-with-hyperlight/
- 开源实现: https://github.com/xingyaoww/code-act
- 实用指南: http://krasserm.github.io/2025/12/16/code-actions/

## 推理层

### 🧠 论文 04: ReflAct — World-Grounded Decision Making via Goal-State Reflection

| 字段 | 内容 |
|------|------|
| 作者 | Jeonghye Kim, Sojeong Rhee 等（KAIST） |
| 发表 | EMNLP 2025 Main Conference |
| 一句话定性 | **v8 的直接灵感来源**：把 ReAct 的自由"thought"替换为"对目标状态的结构化反思" |

**核心贡献**:
- 发现 ReAct 的核心缺陷：`thought` 字段自由度太高 → 累计误差 + 幻觉 → 与真实状态 misaligned
- 提出 ReflAct backbone：每轮 action 前都必须输出 `(current_state, goal_gap, next_step)` 结构化反思
- **在 ALFWorld 基准上达到 93.3% 成功率**，超越 ReAct 27.7%
- 更重要的发现：ReflAct 即使**不加 Reflexion/WKM 等增强模块**，也优于"ReAct + 这些增强模块"——说明**强化核心骨架比叠加外部模块更关键**

**方法要点**（与 ReAct 对比）:
```
ReAct 循环:
  Thought: ???  (free-form)
  Action: ...

ReflAct 循环:
  Current State: 我在 kitchen，手上拿着 knife，goal 是 slice apple
  Goal Gap: 还需要定位 apple；apple 可能在 table 或 fridge
  Next Step: 先去 table
  Action: goto(table)
```

**关键实验**:
- ALFWorld: 93.3% vs ReAct 65.6%
- WebShop: 显著提升
- 消融实验: ReflAct + 无任何增强 > ReAct + Reflexion + WKM

**对本项目启示**:
- v8 的 `_goal_reflect` 直接映射自此论文
- **本次二次评审发现 v8 的 `_get_state_summary` 只返回 TODO 勾选框、不含 result，是对 ReflAct 思想的打折**（见 `v8-goal-driven-planner-review-v2-research-lens.md` 第 3.3 节）

**链接**: https://arxiv.org/abs/2505.15182

### 🧠 论文 05: LATS — Language Agent Tree Search

| 字段 | 内容 |
|------|------|
| 作者 | Andy Zhou, Kai Yan 等（UIUC） |
| 发表 | ICML 2024 |
| 一句话定性 | **首次将 MCTS 引入 LLM Agent 推理**，统一 Reasoning + Acting + Planning |

**核心贡献**:
- 把 ReAct 的**线性轨迹**扩展为**搜索树**：每个节点 = state，每条边 = action
- 引入 **4 个组件**:
  - Selection: UCT 上置信界选择待扩展节点
  - Expansion: LLM 从该节点生成 k 个候选 action
  - Evaluation: Self-Consistency / LLM-as-Judge 打分
  - Backpropagation: 得分回传到祖先节点，更新 UCT 统计
- 在 HumanEval / WebShop / HotpotQA 同时达到 SOTA

**方法要点**:
```
        Root (state_0)
       /    |    \
    a_1    a_2    a_3    <- Expansion (generate k candidates)
    ...    ...    ...
    eval   eval   eval    <- LLM-as-Judge 打分
    0.7    0.4    0.9
     |            |
  backprop update UCT
     |            |
  continue deeper on best branch
```

**关键实验**:
- HumanEval pass@1: 94.4% (vs ReAct 70%)
- WebShop 成功率: 超越之前所有方法

**对本项目启示**:
- 本项目 `dag/` 模块的状态机可作为 LATS 的基础，树只是"受限分支的 DAG"
- **v9 引入 LATS 与 v8 ReflAct 形成天然对照**：
  - v8: 线性反思（1 条轨迹 + 周期性重锚定）
  - v9: 树搜索反思（k 条轨迹 + UCT 选择）
- **融合实验**（本项目可首发的组合）: "每个 LATS 扩展节点都执行一次 ReflAct 式状态反思"

**链接**:
- 论文: https://arxiv.org/abs/2310.04406
- 智源中文解读: https://hub.baai.ac.cn/view/31413

### 🧠 论文 06: DeepSeek-R1 — Incentivizing Reasoning via RL

| 字段 | 内容 |
|------|------|
| 作者 | DeepSeek-AI |
| 发表 | arXiv 2501.12948 (Jan 2025) |
| 一句话定性 | **开源复刻 o1 的里程碑**，证明 outcome-only RL 就能激发 long CoT 推理 |

**核心贡献**:
- R1-Zero: 从 base model 出发、**只用 outcome reward 的 RL**（无 SFT），涌现 self-reflection / verification / "aha moment"
- R1: 引入 cold-start SFT + 多阶段 RL，在数学/代码/推理多基准达到 o1 水平
- 完全开源权重（DeepSeek-R1-Distill-Qwen-7B 等），单机可跑

**方法要点**:
- 奖励模型：accuracy reward + format reward（不用 process supervision）
- 训练中自然涌现"等等，让我重新想一下"这类自反思行为
- 证明"**reasoning 是 RL 激励出来的、不是 prompt 教出来的**"

**对本项目启示**:
- 本项目 Pipeline 范式是"**从外部教 LLM 做反思**"（v8 的 `_goal_reflect`）
- R1 展示了"**让 LLM 自己学会反思**"是可行的
- **v11 建议**: 把 `llm/client.py` 接入 R1 API，加一条 `ENABLE_MODEL_NATIVE` 路径——对**同一个任务**：
  - v8 路径: 外部编排 50 次 LLM 调用
  - Model-native 路径: 1 次调用给 R1（内部做 long CoT + tool use）
- 这个对照是整个教学项目的**压轴章节**

**链接**:
- 论文: https://arxiv.org/abs/2501.12948
- 开源: https://github.com/deepseek-ai/deepseek-r1

### 🤝 论文 07: Chain-of-Agents (CoA) — Long-Context Collaboration

| 字段 | 内容 |
|------|------|
| 作者 | Yusen Zhang, Ruoxi Sun 等（Google Research） |
| 发表 | NeurIPS 2024 (arxiv 2406.02818) |
| 一句话定性 | **Training-free 多 Agent 协作**解决长上下文，超越 RAG 和长 context LLM |

**核心贡献**:
- 把长文档/长任务分段，每段由一个 **Worker Agent** 读取并产出"我从这段里提取的信息块"
- **Manager Agent** 拿到全部块后综合推理、给出最终答案
- 完全 training-free，可扩展到任意长度；在 9 个 QA/summary 基准平均提升 10%+

**方法要点**:
```
[Long Document, 500K tokens]
         ↓ split
[Chunk 1] → Worker_1 → CU_1 (中间信息块)
[Chunk 2] → Worker_2 → CU_2 ← (看到 CU_1)
[Chunk 3] → Worker_3 → CU_3 ← (看到 CU_1,CU_2)
...
[Chunk N] → Worker_N → CU_N
                        ↓
                     Manager → Final Answer
```

**关键实验**:
- NarrativeQA / QuALITY / HotpotQA 长文档子集: 普遍超越 RAG 5-15%
- 成本低于直接塞长 context LLM

**对本项目启示**:
- 本项目 `context/manager.py` 目前用 token budget + truncation 处理长上下文，信息丢失严重
- 本项目 v5/v8 的 TODO 列表可以视为"任务分段" → **每个 TODO 对应一个 Worker agent 执行**
- **v10 建议**: 把 orchestrator 改造成 "Worker pool + Manager" 模式，TODO 并行执行产出 CU，Manager 综合

**链接**:
- 论文: https://arxiv.org/abs/2406.02818
- Google Blog: https://research.google/blog/chain-of-agents-large-language-models-collaborating-on-long-context-tasks/

### 💾 论文 09: MUSE — Experience-Driven Self-Evolving Agent

| 字段 | 内容 |
|------|------|
| 作者 | Fang et al. |
| 发表 | arXiv 2510.08002 (Oct 2025) |
| 一句话定性 | **长时程任务 SOTA**，靠经验驱动的分层记忆实现自进化 |

**核心贡献**:
- 提出三层记忆模块:
  - **Strategic memory**: 高层策略、做哪类任务用哪条路径
  - **Tactical memory**: 中层方法、工具组合套路
  - **Episodic memory**: 具体成功/失败轨迹
- 任务完成后自动触发"经验抽取 + 记忆更新"
- 在 TAC (The Agent Company) 长时程基准上 SOTA

**对本项目启示**:
- 本项目 `memory/long_term.py` 仅存 KV，没有任何"经验抽象"
- **v10 建议**: 扩展为 `memory/strategic.py / tactical.py / episodic.py` 三级，在 `reflector.py` 任务完成后触发抽取
- **对照实验极易设计**: 同一个评测任务连续执行 5 次
  - v8 baseline: 每次都是重头开始
  - v8 + MUSE: 第 2 次起应显著减少轮次 / 避免已知失败路径

**链接**:
- 论文: https://arxiv.org/abs/2510.08002
- HF 讨论: https://huggingface.co/papers/2510.08002
