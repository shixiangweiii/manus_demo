**核心贡献**:
- 三层能力分类：
  - **基础层**（Foundational）：规划 / 工具使用 / 智能搜索（POMDP think-act 结构、ReAct 等）
  - **自我进化层**（Self-evolution）：反馈机制 + 记忆驱动适应（Voyager / Reflexion 等）

- 两条实现路径：
  - 上下文推理（context-time orchestration）
  - 训练后优化（post-training，如 Memory-R1 把推理模式内化进权重）
- 配套开源仓库 *Awesome-Agentic-Reasoning*（数百篇顶会论文 + 应用案例 + 基准测试）

**对 manus_demo 启示**:
- 当前 v1/v2/v5/v8 全部停留在"上下文推理"路径；**短期没必要切换到 post-training 路径**（教学 demo 优先），但应在 README 显性声明定位
- 综述的"基础-进化-协作"三层框架可直接作为我们 `agents/` 目录的章节划分依据

### 📘 论文 03: Toward Efficient Agents — Memory, Tool Learning, and Planning

| 字段 | 内容 |
|------|------|
| 作者 | 上海 AI Lab + 复旦 + 中科院 + 上交大 等 9 校联合 |
| 发表 | 2026（综述）|
| 一句话定性 | **抓痛点的 2026 综述**：直面 token 吞金兽，提出"成本-效果 Pareto"评估范式 |

**核心贡献**:
- 把 Efficient Agent 拆成三大组件：
  - **Efficient Memory**：工作记忆（COMEDY / MemAgent 文本压缩；Activation Beacon / MemoRAG 潜在压缩）+ 外部记忆（向量库 / 知识图谱）
  - **Efficient Tool Learning**：工具选择 / 调用 / 集成推理
  - **Efficient Planning**：单/多智能体协作的低成本规划
- 提出统一报告标准：token 数、调用轮次、延迟、API 成本必须一同披露
- 评估范式：**固定预算比效果 / 同等效果比成本，用 Pareto 前沿看权衡**

**对 manus_demo 启示**（高优先级）:
- 我们 `evaluation/` 目前的 4 维评分（Planning 30 / Execution 40 / Efficiency 20 / Reflection 10）已部分覆盖，但 **Efficiency 维度仅看 trajectory_efficiency × token × time，没显式画 Pareto 前沿**
- 建议在 `evaluation/runner.py` 增加 `cost_effectiveness_curve()` 输出三种模式（simple / complex / emergent）的 Pareto 散点
- LongTermMemory 当前只是 JSON-KV，可考虑借鉴 Activation Beacon 的潜在压缩思路（v10 候选）

**链接**: 解读 https://cloud.tencent.com/developer/article/2637134 ；https://zhuanlan.zhihu.com/p/2014669044825999098

### 🧠 论文 06: Reducing Belief Deviation in RL for Active Reasoning of LLM Agents (T3)

| 字段 | 内容 |
|------|------|
| 作者 | 香港中文大学 + 字节跳动 + Georgia Tech |
| 发表 | **ICLR 2026 Oral**（口头报告，~1-2% 比例） |
| 项目 | https://github.com/unimpor/T3 ；https://openreview.net/forum?id=r8hzDA3pUY |
| 一句话定性 | 用 **belief tracking** 视角解释"多轮 ReAct 越走越偏"，提出 T3（Truncating Belief-Trapped Trajectories）极简修复 |

**问题场景**:
- 在 active reasoning 任务里（信息不完备 + 多轮交互 + 主动获取），随着轮次增加：模型可能不断重复无效动作 / 忽略反馈 / 沿错误判断深入
- 这种偏差在 RL 训练时会污染 credit assignment，导致训练不稳定、探索不足、最终学出"表面会交互、实则不会主动推理"的 Agent

**方法要点**:
- 显式追踪每步对潜在任务状态的"信念"（belief）
- 检测信念漂移（belief deviation）超过阈值的轨迹
- T3：在 RL 训练阶段截断这些 trapped 轨迹，避免污染策略

**对 manus_demo 启示**（高优先级）:
- 我们 `EmergentPlanner` / `ReActEngine` 在长链路下也存在"反复调用同一工具、忽略历史 observation"的退化模式（参考 `MAX_REACT_ITERATIONS=10` 这个保险丝的存在本身就是症状）
- **可在 `react/engine.py` 加一个 belief-deviation 诊断**：每轮记录 LLM 对剩余子目标的判断，超过阈值变化频率即触发"打断 + 重新 plan"
- 不一定要做 RL 训练，**纯推理时的 belief tracking 也能用来当 early-stop 信号**

**链接**: 报道 http://finance.sina.com.cn/wm/2026-04-11/doc-inhuepen6953520.shtml
