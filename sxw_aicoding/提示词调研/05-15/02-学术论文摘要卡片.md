## 论文 1:The Prompt Report(最权威综述)

- **标题**:The Prompt Report: A Systematic Survey of Prompt Engineering Techniques
- **arXiv**:https://arxiv.org/abs/2406.06608
- **更新时间**:2025.09(持续更新)
- **类型**:系统综述

### 核心贡献

汇集了提示工程技术的**完整分类体系**:
- **33 个标准化术语**(给术语下定义,统一行业语言)
- **58 种 LLM 提示技术**(覆盖文本任务全场景)
- **40 种多模态技术**(图像、音频、视频提示)

### 价值

- 行业语言对齐:解决「不同博客术语混乱」问题
- 选型字典:遇到新场景可对照查找已有技术
- 学术起点:做提示工程研究的必备引用

### 推荐阅读姿势

不必通读 ——把它当**工具书 + 索引**用,遇到具体问题时按章节查找对应技术。

## 论文 2:A Systematic Survey of Prompt Engineering in LLMs

- **标题**:A Systematic Survey of Prompt Engineering in Large Language Models: Techniques and Applications
- **arXiv**:https://arxiv.org/abs/2402.07927
- **更新时间**:2025.09
- **类型**:系统综述(覆盖 LLM + VLM)

### 核心贡献

不仅覆盖纯文本 LLM,还扩展到**视觉-语言模型(VLM)** 的提示工程,提供:
- 提示工程在 LLM 与 VLM 的能力扩展中的角色
- 分类综述(任务维度 + 技术维度交叉)
- 应用案例分析

### 与 The Prompt Report 的差异

| | Prompt Report | This Survey |
|---|---|---|
| 重点 | 技术分类全景 | 应用场景 + 多模态扩展 |
| 多模态深度 | 列举 | 系统分析 |
| 适合 | 技术索引 | 落地参考 |

## 论文 3:PE2 — Prompt Engineering a Prompt Engineer

- **标题**:Prompt Engineering a Prompt Engineer
- **arXiv**:https://arxiv.org/abs/2311.05661
- **更新时间**:2025.07
- **类型**:方法论 + 实证

### 核心贡献

提出 **PE2 自动提示工程方法**,通过元提示注入三类要素:
1. **详细描述(Detailed Description)** — 任务目标、约束、输出格式
2. **上下文规范(Context Specification)** — 输入数据特征、典型边界 case
3. **逐步推理模板(Step-by-Step Reasoning)** — 显式推理步骤

### 实测结果

在多个基准测试(BBH、MATH、GSM8K)上,PE2 比基线 APE(Automatic Prompt Engineer)提升 3-15%。

### 工程价值

- 适合「我的 prompt 怎么改进?」场景
- 可与 manus_demo 的 PlannerAgent 提示生成结合 — 用 PE2 自动优化系统提示模板

## 论文 4:A Prompt Pattern Catalog(提示模式目录)

- **标题**:A Prompt Pattern Catalog to Enhance Prompt Engineering with ChatGPT
- **arXiv**:https://arxiv.org/abs/2302.11382
- **更新时间**:2025.02
- **类型**:模式语言(Pattern Language)

### 核心贡献

把提示工程类比为软件设计模式,提出**可复用的提示模式目录**,例如:
- **Persona Pattern**(角色模式)
- **Audience Persona Pattern**(目标受众模式)
- **Recipe Pattern**(配方模式 — 步骤化)
- **Template Pattern**(模板模式)
- **Reflection Pattern**(反思模式)
- **Game Play Pattern**(游戏化模式)

### 学术价值

借鉴 GoF 设计模式语言,把零散的 prompt 技巧抽象为可教学、可组合的「模式」,对工业界 prompt 库建设有直接指导意义。

## 论文 5:PromptFlow(2025 最新)

- **标题**:PromptFlow: A Modular Training Framework for Prompt Engineering
- **arXiv**:https://arxiv.org/abs/2510.12246
- **发布时间**:2025.10
- **类型**:框架 + 实证

### 核心贡献

提出受 **TensorFlow 启发的模块化训练框架** PromptFlow,包含:
- **Meta-Prompt(元提示)** — 模板层
- **Operators(算子)** — 增删改提示组件的原子操作
- **Optimization(优化器)** — 基于梯度的元学习
- **Evaluator(评估器)** — 多任务效果度量
- **强化学习经验回收机制** — 解决「LLM 无法在提示工程过程中积累经验」的问题

### 解决的核心问题

针对现有自动提示工程的两大局限:
1. **静态更新规则** — PromptFlow 引入动态策略选择
2. **整段更新提示** — PromptFlow 支持细粒度提示部分编辑

### 工程价值

适合企业级 prompt 库建设 — 把 prompt 当 ML 模型来训练、版本管理、回归测试。

## 论文 6:LLM 学术论文分析框架(IEEE)

- **标题**:Leveraging LLMs for Research Paper Analysis, Classification and Recommending Conferences
- **IEEE Xplore**:https://ieeexplore.ieee.org/document/11379994/
- **作者**:Poorvi K Joshi 等(印度 Ramdeobaba University)

### 核心贡献

基于 LLaMA-3.1 8B Instruct,使用**高级提示工程技术**实现:
- 学术论文自动分类(Publishable / Non-Publishable)
- 段落级分块的语义评估
- **STORM 架构**(Semantic Topic-Oriented Research Matching)推荐合适会议
- 结构化反馈(优点 / 缺点 / 置信度评分)

### 工程亮点

- **集成 LangChain + Groq API** — 高效文档处理 + 快速推理
- **段落级提示** — 把长文档拆分语义连贯块,而非整篇硬塞
- **快速推理** — LLaMA-3.1 8B 在该任务上实测速度优于更大模型

### 启示

文档级长任务的范式 — 不要试图一次性处理整个文档,**分块 + 段落级提示 + 聚合**才是工程现实。
